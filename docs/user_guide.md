# A股全市场资金分析系统 - 用户使用手册

本文档介绍如何日常运行和使用本系统，以捕捉主力资金动向并规避风险。

## 1. 日常操作流程

本系统设计为 **"每日收盘后更新 -> 次日开盘前分析"** 的工作模式。

### 步骤一：数据更新 (每日必做)
**执行时间:** 建议每日 **24:00 (午夜)** 或 **次日 08:00 前**。
**目的:** 从互联网拉取最新的行情、两融和北向资金数据，并进行清洗入库。

**操作命令:**
```bash
cd D:\cc_codex\stock_g
python data_loader.py
```

*   **耗时:** 日常增量更新约需 3-5 分钟（视网络情况而定）。
*   **注意:** 该脚本会自动处理沪深两市 5000+ 只股票，包含重试机制。如果网络中断，重新运行即可。

---

### 步骤二：启动分析界面与实时盯盘
**执行时间:** 交易日 **09:30 - 15:00**。

**操作:** 点击左侧边栏的 **"🔄 刷新界面"**。
**效果:** 系统将保留昨晚的分析信号，同时实时抓取当前的：
*   **现价 & 涨跌%**: 观察信号个股当前是否在异动拉升。
*   **今开%**: 识别主力早盘态度（高开代表强势）。
*   **量比**: 识别是否有真钱在买。

---

### 步骤三：模拟实战 (训练盘感)
**执行时间:** 盘中或复盘时。

**操作:** 导航至 **"🎮 模拟炒股"** 页面。
1.  **初始账户**: 系统为您分配了 100,000 元虚拟资金。
2.  **下单**: 输入 6 位股票代码，确认现价后输入数量（100的倍数），点击买入。
3.  **持仓**: 页面会自动按最新价计算您的“浮动盈亏”和“收益率”。
4.  **复盘**: 观察自己根据信号买入的股票，在 3-5 天内的表现，不断修正自己的选股逻辑。

## 2. 界面功能解读

### A. 市场概览 (Market Overview)
这是系统的首页，用于判断**"今天要不要出手"**。

1.  **大盘情绪温度计:**
    *   **🔥 狂热:** 上涨家数 > 80%。警惕追高风险。
    *   **🔴 温和:** 上涨家数 > 50%。市场环境良好，适合操作。
    *   **🟢 低迷 / ❄️ 冰点:** 上涨家数 < 20%。空头排列，建议**空仓**或轻仓试错。

2.  **资金榜单 (Top 10):**
    *   **融资净买入强度榜:** 内资主力（杠杆客）最看好的股票。
    *   **北向资金扫货榜:** 外资（聪明钱）买入最多的股票。
    *   *技巧: 如果一只股票同时出现在两个榜单上，那是"双重共振"，值得重点关注。*

### B. 智能选股 (实时验证版)
在选股列表中，除了看信号，重点关注以下实时组合：
*   **信号 (强力买入) + 量比 (> 1.5)**: 极高概率的主升浪起点。
*   **信号 (强力买入) + 涨跌% (< 0)**: 可能是昨晚利好今日回调，适合低吸。
*   **今开% (< -2%)**: 即使昨天信号再好，也要警惕“不及预期”，建议等企稳。

### C. 个股深度分析 (Deep Dive)
输入股票代码（如 `600519` 或 `000858`），查看详细图表。

*   **K线图:** 叠加了 MA20 生命线。
*   **融资区域图 (橙色):** 观察融资余额是否在不断攀升（主力吸筹）。
*   **北向曲线 (蓝色):** 观察外资持仓是否在不断创新高。

## 3. 信号含义说明

系统根据资金流向和技术形态自动打标：

| 信号图标 | 名称 | 含义与操作建议 |
| :--- | :--- | :--- |
| 🟢 | **强力买入** | **主力+外资+趋势共振**。胜率最高的信号，建议重点关注。 |
| 🟡 | **关注 (加速)** | 融资大幅买入且放量，短线爆发力强。 |
| 🟡 | **关注 (反弹)** | 资金抄底但趋势仍向下（左侧交易）。风险较高，博反弹用。 |
| ⚫ | **止损离场** | 跌破生命线且资金流出。坚决卖出，不要犹豫。 |
| 🔴 | **减仓** | 涨幅过大，获利盘丰厚，建议落袋为安。 |
| ⚫ | **黑名单** | **ST股 / 微盘股 / 严重亏损**。系统自动拦截，**严禁买入**。 |

## 4. 常见问题 (FAQ)

**Q: 为什么我看不到当天的实时信号？**
A: 本系统基于**日线收盘数据**。盘中看到的信号是基于**昨天收盘**分析得出的结论。盘中建议结合实时分时图操作。

**Q: 为什么有些股票数据是空的？**
A: 可能是新股，或者该股票不是融资融券标的（两融标的只有约 3000 只，而非全市场）。

**Q: 如何备份数据？**
A: 直接复制项目目录下的 `stock_data.db` 文件即可。
